10// Example random big-small scheduler using thread affinity
11//
12// The hardware consists of B big cores and S small cores
13// This scheduler selects B threads that have affinity to (all) big cores,
14// while the other threads have affinity to the small cores.
15// Periodically, or when threads on a big core stall/end,
16// new threads are promoted to run on the big core(s)
17//
18// This scheduler uses only setThreadAffinity(), and leaves all messy
19// low-level details that govern thread stalls etc. to the implementation
20// of SchedulerPinnedBase
21

30 m_num_big_cores = 0;

141

145 {

150 }
151

153 {
154 // Randomly select a thread from our list
